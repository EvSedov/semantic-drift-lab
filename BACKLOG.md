# Backlog

Короткий список следующих шагов по проекту. Это не roadmap в строгом смысле, а рабочий backlog: что стоит сделать дальше, когда появится время и мотивация.

## Архитектура

- Оформить проект в модели `library core + CLI shell`.
  Сейчас ядро уже живёт в `semantic_drift_lab`, а `run.py` выступает как пользовательский вход. Следующий шаг — сделать это более явным на уровне packaging и entrypoint.

- Добавить нормальный CLI entrypoint поверх пакета.
  Варианты: `python -m semantic_drift_lab` или `console_scripts`, чтобы запуск не зависел от отдельного `run.py`.

- Продолжить отделять пользовательский интерфейс от аналитической логики.
  CLI должен оставаться тонкой оболочкой, а вся предметная логика — жить в библиотечном слое.

## Поиск по документам

- Обновить README под новые режимы `--explain-search` и `no_results`.
  Сейчас код уже умеет объяснять ранжирование и отдельно обрабатывать случай, когда по запросу ничего не найдено; это стоит явно зафиксировать в документации.

- Подумать о более явной обработке низкоинформативных запросов.
  Уже есть отсечение слишком коротких запросов и части стоп-слов. Если понадобится, можно развивать это дальше, но без избыточной магии.

## Упаковка и удобство использования

- Добавить packaging-метаданные проекта.
  Например, `pyproject.toml`, чтобы проект было проще устанавливать и запускать как нормальный Python-пакет.

- Решить судьбу `run.py`.
  Либо оставить его как совместимый thin wrapper, либо со временем заменить на официальный entrypoint пакета.

## Качество и сопровождение

- Добавить минимальные smoke-тесты для CLI и doc-search.
  Особенно полезно зафиксировать кейсы:
  `no_results`, `--explain-search`, короткие запросы, токенизация `ё/Ё`.

- Подготовить небольшой стабильный набор тестовых данных для регрессий.
  Это поможет безопаснее менять эвристики поиска и пороги доверия.
