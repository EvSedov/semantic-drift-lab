# Semantic Drift Lab

**Semantic Drift Lab** — учебный и исследовательский Python-проект для анализа текстовых записей, семантического сходства и временных рядов через классические математические методы, без нейросетей и без обучения моделей.

Проект собран как независимый, детерминированный пайплайн, вдохновлённый идеей геометрического анализа корпуса: представлять наблюдения в компактном пространстве, выделять устойчивые структуры и отслеживать дрейф поведения системы во времени.

Поиск по markdown-документам в этом репозитории не является центральной идеей проекта. Это только один из прикладных режимов поверх более общего ядра для анализа текстового корпуса и числового сигнала.

## Зачем этот проект

Этот репозиторий нужен не как production-сервис, а как площадка для экспериментов и понимания нескольких идей сразу:

- как превратить набор текстовых записей в компактные числовые представления;
- как искать похожие записи без эмбеддингов от внешних моделей;
- как отличать устойчивые паттерны от шумовых точек;
- как анализировать динамику качества по временной последовательности;
- как собрать всё это в прозрачный и воспроизводимый CLI-инструмент.

Если коротко: это исследование того, что можно получить из данных с помощью `TF-IDF`, `SVD`, `Takens embedding`, `kNN stability` и `Kalman filter`, когда хочется понимать математику, а не прятать всё внутри чёрного ящика.

## Что делает пайплайн

Проект объединяет четыре шага в один детерминированный поток:

| Шаг | Метод | Назначение |
|------|--------|------------|
| 1 | `TF-IDF + TruncatedSVD` | превращает тексты в компактные векторы |
| 2 | `kNN Stability` | оценивает, какие точки устойчивы, а какие похожи на шум |
| 3 | `Takens Embedding` | восстанавливает фазовое пространство по числовому ряду |
| 4 | `Kalman Filter` | сглаживает ряд и отмечает возможный дрейф |

На практике это позволяет:

- находить похожие записи по смыслу;
- выделять более устойчивые наблюдения, которые можно интерпретировать как локальные "аттракторы";
- анализировать числовой сигнал как динамическую систему;
- замечать аномальные отклонения или ухудшение поведения системы.

## Для каких данных он задуман

Основной сценарий проекта — анализ коллекции текстовых записей и, при наличии, связанного с ними числового сигнала.

Типичный объект может содержать:

- `label` или `title`
- `text`, `content` или `description`
- `signal`, `score`, `value` или другой числовой показатель
- произвольные метаданные

Из текстовых полей собирается единый текст для SVD-представления, а числовое поле используется как временной ряд для Takens и Kalman.

Проект можно развивать как лабораторию для любых наборов записей, где есть:

- текстовое описание наблюдения;
- метаданные;
- опциональный числовой показатель, меняющийся во времени.

## Как это устроено

### 1. SVD-представление текстов

Модуль [`semantic_drift_lab/svd_embed.py`](./semantic_drift_lab/svd_embed.py) строит матрицу `TF-IDF`, затем уменьшает размерность через `TruncatedSVD`, после чего нормализует векторы.

Это даёт:

- компактное пространство признаков;
- косинусное сходство между записями;
- базу для поиска похожих задач и локальной геометрии корпуса.

### 2. Оценка устойчивости через kNN

Модуль [`semantic_drift_lab/knn_stability.py`](./semantic_drift_lab/knn_stability.py) слегка возмущает точки и смотрит, насколько сохраняется состав ближайших соседей.

Идея простая:

- если точка остаётся рядом примерно с теми же соседями, она стабильна;
- если соседи постоянно меняются, точка относится к шумовой или переходной области.

Это даёт численный `stability score` в диапазоне `[0, 1]`.

### 3. Takens embedding для временного ряда

Модуль [`semantic_drift_lab/takens_embed.py`](./semantic_drift_lab/takens_embed.py) преобразует одномерный ряд в delay-векторы.

Здесь числовой ряд рассматривается не просто как список значений, а как след скрытой динамики системы. Это полезно для исследовательского анализа, когда интересно не только "сколько", но и "как система движется".

### 4. Kalman drift detection

Модуль [`semantic_drift_lab/kalman_drift.py`](./semantic_drift_lab/kalman_drift.py) применяет простой одномерный фильтр Калмана к числовому ряду.

На выходе получаются:

- сглаженный ряд;
- инновации;
- флаги точек, где отклонение выглядит аномальным;
- итоговый `drift_score`.

## Режимы работы

Основной интерфейс проекта — CLI в [`run.py`](./run.py).

Основной Python API теперь тоже идёт через `semantic_drift_lab`.

```python
from semantic_drift_lab import SemanticDriftPipeline, CorpusRecord

records = [
    CorpusRecord(idx=0, label="task a", text="alpha beta gamma", signal=4.0),
    CorpusRecord(idx=1, label="task b", text="beta gamma delta", signal=5.0),
]

result = SemanticDriftPipeline(svd_components=2).run_records(records)
print(result.embeddings.shape)
```

### Полный анализ JSONL

```bash
python run.py
```

По умолчанию проект пытается прочитать файл:

```text
./data.jsonl
```

Можно указать свой входной файл:

```bash
python run.py --input /path/to/data.jsonl
```

### Поиск похожих записей по произвольному запросу

```bash
python run.py --input /path/to/data.jsonl --find-similar "ошибка в пайплайне индексации"
```

### JSON-вывод без графиков

```bash
python run.py --input /path/to/data.jsonl --json --pretty
```

### Поиск по markdown-документам

Отдельный дополнительный режим умеет индексировать markdown-файлы в произвольной директории и искать по ним похожие документы:

```bash
python run.py --doc-query "kalman drift" --doc-path ~/documents
```

При повторных запусках индекс поднимается из кэша:

```text
~/.cache/semantic-drift-lab/markdown_corpus_index.pkl
```

Если нужно пересчитать индекс вручную:

```bash
python run.py --doc-query "takens" --doc-path ~/documents --rebuild-index
```

## Что получается на выходе

В обычном режиме проект печатает консольный отчёт и создаёт визуализации:

- `output/clusters.png` — проекция SVD-пространства с окраской по `kNN stability`;
- `output/drift.png` — наблюдения, Kalman smoothing и инновации.

Отчёт включает:

- долю объяснённой дисперсии SVD;
- список наиболее устойчивых записей;
- похожие записи для каждой записи;
- drift alert по числовому ряду;
- форму и примеры Takens-векторов.

## Почему проект может быть полезен для обучения

Этот репозиторий хорош для учебных целей, если хочется:

- увидеть реальный, но небольшой аналитический пайплайн целиком;
- понять, как текстовая аналитика связывается с анализом временных рядов;
- поэкспериментировать с гиперпараметрами без долгого цикла обучения;
- исследовать интерпретируемые методы вместо нейросетевого black box;
- использовать код как базу для собственных экспериментов.

Особенно полезно то, что логика здесь прозрачная: почти каждый шаг можно открыть, прочитать и быстро понять.

## Ограничения

Важно воспринимать этот проект именно как исследовательский:

- это не строгая научная репликация оригинального proprietary DEMON;
- качество результатов сильно зависит от входных текстов и стабильности метрик;
- числовой сигнал может быть лишь приближённой метрикой и не всегда отражает реальное качество системы;
- здесь нет обученной семантической модели, поэтому поиск сходства ограничен возможностями `TF-IDF + SVD`;
- в репозитории пока нет тестового набора и автоматизированной валидации качества.

## Что можно обобщать дальше

Проект уже сейчас ближе к универсальной исследовательской заготовке, чем к инструменту только для одного конкретного домена. Следующие естественные шаги развития:

- отделить ядро анализа от загрузчиков конкретных источников данных;
- развивать и стабилизировать общий формат записи;
- сделать markdown search отдельным адаптером, а не частью основного нарратива;
- разрешить запуск только текстового анализа без временного ряда;
- добавить другие источники данных: CSV, plain text corpus, logs, notes.

Часть этого движения уже началась: markdown-поиск и загрузка JSONL вынесены в адаптеры, а ядро работает с общим форматом записи.

## Связь с оригинальным DEMON

Оригинальный DEMON, на который проект концептуально опирается, описывается как proprietary-подход без открытого peer-reviewed корпуса реализации.

Этот репозиторий не претендует на буквальное воспроизведение оригинала. Он делает другое:

- берёт схожую исследовательскую интуицию;
- собирает её из открытых и понятных математических методов;
- адаптирует их к анализу текстовых записей и числовых сигналов в локальном Python-пайплайне.

## Коротко

**Semantic Drift Lab** — это компактный исследовательский проект о том, как анализировать текстовые записи, семантическое сходство и динамику числового сигнала через прозрачные математические методы. Он подходит как для практических локальных экспериментов, так и для обучения: читать код, менять параметры, наблюдать, как меняется поведение пайплайна, и лучше понимать сами идеи.
